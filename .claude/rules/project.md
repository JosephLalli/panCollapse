# Project rules

- `docs/product-spec.md` is the canonical contract.
- Do not change settled behavior without an ADR entry and human approval.
- Do not create production code until Gate Architecture Approved has passed.
- Do not broaden V1 with velocity state, synthetic targets, BAM/BUS output, automatic
  closest-copy inference, or a custom persistent lookup index.
- Update `PROGRESS.md` and the decision log after substantive work.
